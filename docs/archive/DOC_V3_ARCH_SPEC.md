# AI-Burp V3: 架构规格说明书
## 自主红队智能体基础设施

## 1. 设计哲学: "Triple-A" 标准
AI-Burp V3 被设计为 AI 智能体的**红队操作系统**。它遵循 Triple-A 标准：
- **Asynchronous (异步)**: I/O 零等待，用于大规模攻击面发现。
- **Autonomous (自主)**: 具有内部决策树的自动驾驶扫描器逻辑。
- **Agentic (智能体化)**: 旨在作为大语言模型的“手和眼”。

## 2. 核心组件架构

### 2.1 异步引擎 (`burp_v3.py`)
V3 的核心是一个基于 `httpx.AsyncClient` 构建的事件循环驱动请求器。
- **流控**: 使用 `asyncio.Semaphore` 维持每个域名的精确并发限制。
- **自动打标**: 每一个请求在发出前都会经过 `IntentAnalyzer` 中间件，将语义元数据附加到 `Response` 对象。

### 2.2 智能层 (`intel.py`)
该层代表了智能体的“长期记忆”。
- **KnowledgeBase (KB - 知识库)**: 一个持久化的、以 JSON 为后端的存储库，用于跟踪 `Assets` (资产)。一个资产不仅仅是一个 URL，而是一个具有上下文的发现（例如：JWT secret，内网主机名，或特定的 WAF 绕过技术）。
- **VulnerabilityChainer (漏洞链串联器)**: 一个查询知识库的逻辑引擎。它通过识别不同发现之间的依赖关系（例如：发现 A + 漏洞 B = 关键链路 C）来完成“攻击路径合成”。

### 2.3 语义分析器 (`burp_v3.py`)
V3 使用词法模式匹配引擎来推断端点的**业务意图 (Business Intent)**。
- **映射**: 将 URL/参数模式匹配到权重矩阵。
- **优先级**: 决定 `AsyncDetectors` (异步探测器) 的执行顺序。高价值目标（如 `#AUTH`）会尽早触发激进的绕过逻辑。

## 3. 漏洞探测器 (`detectors_v3.py`)
探测器不再是顺序执行的 Shell 脚本。它们是**并发协程 (concurrent coroutines)**。
- **批量测试**: 在单个 `SQLiDetector.detect()` 中，基线测试、错误探测和基于时间的休眠测试通过 `asyncio.gather` 并行执行。
- **置信度评分**: 探测器返回具有置信度级别的 `Finding` 对象。“高”置信度的发现会自动更新 `KnowledgeBase`。

## 4. AI 原生 API 参考

### `AsyncSmartBurp.smart_scan(url, param, value)`
AI 智能体的主要入口点。它封装了整个侦察和检测全生命周期。
- **输出**: 一个 `Decision` 对象。
- **逻辑**: 
    1. 语义分析识别参数的“性质”。
    2. 并发扫描触发相关的探测器。
    3. 结果与 `KnowledgeBase` 匹配以进行链式攻击。
    4. 为智能体生成战略报告。

## 5. 安全与隐蔽性架构
- **JA3 指纹随机化**: 支持动态 TLS 指纹识别，以模拟多样的浏览器环境。
- **自适应速率限制**: 引擎监控响应头（如 `Retry-After`）和状态码 (429/403)，在无需人工干预的情况下进行自限流。
- **WAF 行为画像**: 当检测到 WAF 模式时，决策会暂停，允许 AI 从 `Bypass` Payload 库中选择具体的编码/绕过策略。
